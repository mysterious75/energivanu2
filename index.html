1. Architecture Overview

Understand how machine learning outputs connect with optimal BESS dispatch action

Energivanu operates on a hierarchical optimization framework, combining machine learning predictions with mathematical programming control:

Telemetry Logs (Power, Temp, Util)
↓
Data Processor (data.py)
↓
EnergivanuPEB (model.py)
↓
Regression (Power Forecast)
Classification (Dispatch Command)
↓
MPCController (mpc.py)
↓
Optimal BESS Battery Dispatch Action

A. Neural Sequence Prediction

The core ML module model.py implements the dual-head EnergivanuPEB architecture comprising a Temporal Convolutional Network (TCN) backbone coupled with Multi-Head Self-Attention. It includes per-feature normalizers, causal convolutions, and predicts multi-step power regressions alongside a 3-class BESS signal.

B. Model Predictive Control (BESS Smoothing)

The mpc.py controller MPCController minimizes grid power deviation against capacity targets:

Min Cost = Sum_{k=1}^N [ Q * (P_grid,k - P_target)^2 + R * u_k^2 + S * (u_k - u_{k-1})^2 ]

Where u is the BESS power action, Q penalizes grid deviation, R preserves battery health, and S limits ramping rates.

C. Peak Shaving & Phase Staggering

The toolkit also implements a hierarchical PeakShavingOptimizer designed to match commercial Time-of-Use tariffs, and a PhaseStaggeringScheduler that staggers GPU sync phases (All-Reduce execution steps) to mitigate cumulative load spikes.

B. Python Quick Start

from energivanu import MPCController, PhaseStaggeringScheduler
import numpy as np

# Initialize MPC controller
mpc = MPCController()
trace = np.sin(np.linspace(0, 4*np.pi, 30)) * 50 + 200
result = mpc.simulate(trace)

print(f"Smoothing percentage: {result['metrics']['smoothing_percentage']}%")

C. Legal & Compliance Diligence

⚠️ Legal & Compliance Diligence

Safe-by-Default Compliance: The pre-trained weights distributed in this repository (best_model_demo.pt) are trained purely on synthetic data. To prevent licensing violations of York University H100 CC BY-NC-ND data, real-data training checkpoints are not distributed. Real-data training scripts (train_real.py) are executed locally by users on their private telemetry clusters.

Scale & Validation Disclaimer: BESS grid smoothing, peak demand shaving, and phase-staggering scheduling metrics are verified against synthetic load traces and scheduled offsets. The 1.85% MAPE prediction error is the only metric evaluated on real GPU telemetry data, which was measured on a single 8-GPU H100 node. Scaling any of these results to a 100K+ GPU facility footprint is a mathematical projection and has not been empirically verified in a live production environment.
